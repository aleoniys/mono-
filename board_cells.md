# Клітинки поля (0–39)

**Гра читає дані з файлу `board_cells.json`.** Щоб змінити ціни, оренду, залог, викуп або вартість зірок — правь лише `board_cells.json`; backend і фронтенд підхоплять значення автоматично.

## Загальні правила
- **Компанії:** ціна 1000. Оренда **прописана явно** за кількістю зірок: 0⭐ — одна сума, 1⭐ — інша, 2⭐ — інша, 3⭐ — інша (без формули; значення в таблиці нижче). Залог 500, викуп 1000. Покращення 500 за зірку, продаж зірки +500.
- **Авто (4, 14, 24, 34):** ціна 500. Оренда за кількість авто у власника: 1 авто = 200, 2 = 400, 3 = 600, 4 = 1000. Залог 250, викуп 500. Покращень немає.
- **Кути та Шанс:** не купуються.

## Опис полів

| Поле | Змінна (JSON) | Що означає |
|------|----------------|------------|
| **№** | `id` | Номер клітинки на полі (0–39). |
| **Назва** | `name` | Назва клітинки (компанія, авто, кут або «Шанс»). |
| **Тип** | `type` | **company** — компанія; **car** — авто; **corner** — кут; **chance** — Шанс. |
| **Ціна** | `price` | Скільки балів потрібно заплатити банку, щоб купити клітинку. Для кутів і Шансу — `null`. |
| **Оренда 0⭐** | `rent_0` | Оренда без покращень (0 зірок). У **авто** — оренда за 1 авто у власника. У компаній — просто прописана сума, без множення. |
| **Оренда 1⭐** | `rent_1` | Оренда з 1 зіркою. У **авто** — оренда за 2 авто у власника. |
| **Оренда 2⭐** | `rent_2` | Оренда з 2 зірками. У **авто** — оренда за 3 авто у власника. |
| **Оренда 3⭐** | `rent_3` | Оренда з 3 зірками. У **авто** — оренда за 4 авто у власника. |
| **Залог** | `mortgage` | Скільки балів дає банк при закладенні клітинки. Викуп — протягом 10 ходів. |
| **Викуп** | `unmortgage` | Скільки балів потрібно заплатити банку, щоб зняти клітинку з залогу. |
| **Покращення (1/2/3⭐)** | `upgrade_cost` | Вартість однієї зірки покращення. Тільки для компаній; у авто — `null`. |
| **Продаж зірки** | `sell_star` | Скільки балів дає банк за продаж однієї зірки. Тільки для компаній. |

### Приклад запису (клітинка «Кіоск») — що означає кожне поле

```json
{
  "id": 1,
  "name": "Кіоск",
  "type": "company",
  "price": 800,
  "rent_base": 50,
  "rent_0": 50,
  "rent_1": 150,
  "rent_2": 250,
  "rent_3": 350,
  "mortgage": 500,
  "unmortgage": 1000,
  "upgrade_cost": 500,
  "sell_star": 500
}
```

| Поле | Значення | Що означає |
|------|----------|------------|
| `id` | 1 | Номер клітинки на полі (перша після Старту). |
| `name` | "Кіоск" | Назва, яка показується на клітинці та в модалках. |
| `type` | "company" | Тип: компанія (можна купувати і ставити зірки). |
| `price` | 800 | Ціна покупки: гравець платить банку **800** балів, щоб купити цю клітинку. |
| `rent_base` | 50 | Базова оренда (лише для довідки; гра використовує `rent_0`…`rent_3`). |
| `rent_0` | 50 | Оренда при **0 зірок**: супротивник платить **50** балів, ставши на цю клітинку. |
| `rent_1` | 150 | Оренда при **1 зірці**: супротивник платить **150** балів. |
| `rent_2` | 250 | Оренда при **2 зірках**: супротивник платить **250** балів. |
| `rent_3` | 350 | Оренда при **3 зірках**: супротивник платить **350** балів. |
| `mortgage` | 500 | **Залог:** гравець отримує від банку **500** балів, закладаючи клітинку (потім має викупити протягом 10 ходів). |
| `unmortgage` | 1000 | **Викуп із залогу:** щоб зняти клітинку з залогу, потрібно заплатити банку **1000** балів. |
| `upgrade_cost` | 500 | **Покращення:** кожна нова зірка коштує **500** балів. |
| `sell_star` | 500 | **Продаж зірки:** банк повертає **500** балів за одну зняту зірку. |

Додатково в JSON: `rent_base` — для довідки; `rent_note` — примітка для авто. На рівні файлу: `upgrade_cost_per_star`, `sell_star_value`. Усі зміни внось лише в `board_cells.json`.
